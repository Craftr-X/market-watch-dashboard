## 1. 产品概述

### 1.1 背景与问题

当前产品 **market-watch-dashboard**（A股每日行情与强势观察）已具备以下能力：

- 每日收盘后自动刷新全市场快照（指数 / 板块 / 个股）
- 基于评分算法筛选出"强势股 Top 20"
- 风险雷达识别 ST / 停牌 / 涨幅异常等样本
- 每日复盘摘要自动生成

**现有局限**：强势股 Top 20 是一张"今日快照"，仅能看到"今天谁强"。用户无法判断：

- 一只强势股是"今天放量脉冲"还是"趋势已经在起"
- 近 3 个月它的走势是否稳健，是否在走主升浪
- 与大盘（沪深 300 / 创业板指）相比，它是相对强势还是随波逐流
- 它的量能是否在逐步放大，有没有吸筹迹象

**本质问题**：产品目前是"市场横向扫描仪"，缺少"个股纵向时间轴"，用户无法在时间维度上验证自己的判断。

### 1.2 目标与价值

本需求的目标是：为产品新增一个**个股历史走势分析模块**，让用户在选定任意 A 股后，能在同一个金融终端界面内：

1. 查看**日线 / 周线 / 月线**三种周期的价格走势折线图
2. 叠加 **MA5 / MA10 / MA20 / MA60** 均线，直观看趋势
3. 下方显示**成交量柱状图**，识别放量与缩量时机
4. 一键**前复权 / 不复权**切换，消除除权除息的视觉跳空
5. 在现有的强势观察 Top 20 表中**点击任意股票代码**直接跳转详情页
6. 支持**股票代码 + 名称模糊搜索**，快速定位目标

**产品价值**：

| 维度 | 现状 | 升级后 |
|------|------|--------|
| 数据深度 | 今日快照 | 日 / 周 / 月历史走势 |
| 趋势判断 | 靠猜 | 有均线辅助 |
| 量能分析 | 无 | 副图成交量柱 |
| 用户路径 | 只能看整体 | 可深入研究个股 |
| 投资辅助 | 粗筛 | 细粒度复盘 |

### 1.3 与现有系统的关系

本模块是**现有 dashboard 的横向扩展**，不是替代。关系如下：

```
┌──────────────────────────────────────────────────────┐
│                   market-watch-dashboard            │
├──────────────────────┬──────────────────────────────┤
│   首页（已有）         │   个股详情页（新增）           │
│   · 指数概览          │   · 个股 K 线 / 折线图         │
│   · 板块热度          │   · MA 均线叠加               │
│   · 强势 Top20 ←──────┼───→ 点击代码跳转个股详情        │
│   · 风险雷达          │   · 成交量副图                 │
│   · 每日复盘          │   · 周期切换（日/周/月）        │
│                      │   · 复权方式切换               │
│                      │   · 股票搜索栏                 │
└──────────────────────┴──────────────────────────────┘
                    ↑共用后端 API 与 SQLite 存储
```

### 1.4 成功标准（OKR）

- **Objective 1**：用户能在 3 步操作内（首页点击 / 搜索 / 跳转）看到任意 A 股的日 / 周 / 月走势图
- **Objective 2**：首次冷启动（某只股票从未被查询过）数据加载时间 ≤ 60 秒；缓存命中时 ≤ 2 秒
- **Objective 3**：图表能清晰展示 MA5 / MA10 / MA20 / MA60 四条均线，并支持前复权与不复权一键切换
- **Objective 4**：代码风格、数据结构、路由命名与现有项目保持一致，不引入技术债务

## 2. 用户分析与使用场景

### 2.1 目标用户画像

| 维度 | 描述 |
|------|------|
| 技术背景 | 非程序员，有基本电脑操作能力，能看懂 K 线图基本概念 |
| 投资阶段 | 有一定投资经验，知道什么是均线、量比、复权；不是纯新手 |
| 使用频率 | 每日收盘后 1~2 次（复盘）；盘中偶尔查 1~2 只 |
| 核心诉求 | 想在"今日谁强"之外，看到"这只股票的趋势怎么样" |
| 使用设备 | 主要 PC 端浏览器，偶尔手机查看 |
| 心理预期 | 不期望实时数据（知道有 15~30 秒延迟）；不期望买卖信号 |

### 2.2 典型使用场景

**场景 1 — 复盘验证**
> 用户在首页看到"宁德时代"出现在强势 Top 5，点进去看月线图，发现近 6 个月高点不断抬高、均线多头排列，成交量逐月放大，判断这不是"一日游"行情，而是趋势在起。关闭页面，继续坚持自己的定投计划。

**场景 2 — 排雷检查**
> 用户搜索一只冷门股，看到它的日线图在某个位置出现了连续跌停后的"钓鱼线"形态（放量冲高后快速回落），结合风险雷达的提示，判断该公司有潜在问题，放弃关注。

**场景 3 — 大盘对比**
> 用户想看"中国平安"这段时间相对沪深 300 是跑赢还是跑输。先看个股日线，再去首页看沪深 300 指数图，人工对比两者的相对强弱（系统暂不提供自动对比功能，作为后续增强项）。

**场景 4 — 均线趋势判断**
> 用户输入"000001"（平安银行），切到周线，看到 MA10 从下往上穿过 MA20（均线金叉），结合成交量放大，判断可能有中期趋势机会。但产品仍标注"本系统仅用于学习，不构成投资建议"。

### 3.2 P0 功能详细说明

#### 3.2.1 个股详情页 — 路径与参数

**URL 格式**：`/stock/[code]?period=daily&adjust=qfq`

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `code` | string（6位数字） | 必填 | `600519` / `000001` / `300750` 等 | 股票代码 |
| `period` | string | `daily` | `daily` / `weekly` / `monthly` | 周期 |
| `adjust` | string | `qfq` | `qfq` / `none` | 复权方式：`qfq` = 前复权，`none` = 不复权 |

**路由示例**：
- `/stock/600519` → 默认显示茅台日线前复权
- `/stock/000001?period=weekly` → 显示平安银行周线前复权
- `/stock/300750?period=monthly&adjust=none` → 显示宁德时代月线不复权

**进入路径**：
1. 首页强势 Top20 表格 → 点击股票代码 → 跳转 `/stock/[code]`
2. 顶部搜索框 → 输入代码或名称 → 跳转 `/stock/[code]`
3. 直接在浏览器输入 URL

---

#### 3.2.2 图表核心 — 图表类型与数据映射

**采用图表库**：lightweight-charts v4（TradingView 出品）

**图表类型**：主图用**K 线（candlestick）**（可兼作折线用），下方副图用**柱状图（histogram）**。

> 为什么选 K 线而不是纯折线？
> K 线天然包含开/收/高/低四价信息，用户可以同时看到实体大小（涨跌力度）和上下影线（盘中博弈）。在技术分析场景中，K 线的可读性远高于纯收盘价折线。

**K 线字段映射**：

| K 线字段 | 说明 | 来源 |
|----------|------|------|
| `time` | 时间戳（UTC，YYYY-MM-DD） | AkShare `日期` |
| `open` | 开盘价 | AkShare `开盘` |
| `high` | 最高价 | AkShare `最高` |
| `low` | 最低价 | AkShare `最低` |
| `close` | 收盘价 | AkShare `收盘` |

**成交量副图字段映射**：

| 副图字段 | 说明 | 来源 |
|----------|------|------|
| `time` | 时间戳（UTC） | 同 K 线 |
| `value` | 成交量（手） | AkShare `成交量` |
| `color` | 红色（上涨）/ 绿色（下跌） | 根据当日 `close >= open` 判断 |

**均线计算规则**（前端实时计算，不存库）：

| 均线 | 计算方式 | 颜色（默认） |
|------|----------|-------------|
| MA5 | 收盘价的 5 日简单移动平均 | 白色 |
| MA10 | 收盘价的 10 日简单移动平均 | 黄色 |
| MA20 | 收盘价的 20 日简单移动平均 | 紫色 |
| MA60 | 收盘价的 60 日简单移动平均 | 绿色 |

> **注意**：均线只在日线周期下显示完整；周线和月线的 MA20 / MA60 由于数据点稀少（周线 1 年约 50 根，月线 1 年约 12 根），默认只显示 MA5 / MA10，不显示 MA20 / MA60。如用户有特殊需求，可在 UI 上标注"数据点不足"。

---

#### 3.2.3 周期切换

**交互方式**：详情页顶部有一排周期切换按钮（日 / 周 / 月），点击后：
1. 立即替换 URL 参数 `period`
2. 显示 Loading 骨架
3. 请求新数据（走缓存则直接展示）
4. 图表平滑重绘

**数据量约定**：

| 周期 | 默认回溯时间 | 约合数据点数 | 图表渲染目标 |
|------|------------|------------|------------|
| 日线 | 最近 3 年 | ~720 根 K 线 | < 500ms |
| 周线 | 最近 5 年 | ~260 根 K 线 | < 300ms |
| 月线 | 最近 10 年 | ~120 根 K 线 | < 200ms |

> 日线默认拉 3 年而非 5 年的原因：超过 3 年后，前复权数据容易出现"历史价格接近 0"的问题（尤其是上市初期股价低的股票），且数据量大影响首屏速度。

---

#### 3.2.4 复权方式切换

**交互方式**：详情页右上角有两个切换按钮（前复权 / 不复权），互斥选中态。

**前端行为**：
- 切换后，重新请求数据（带新的 `adjust` 参数）
- 不在本地做换算（AkShare 接口原生支持 `adjust` 参数）

**视觉差异说明**（tooltip 中提示用户）：

| 模式 | 优点 | 缺点 |
|------|------|------|
| 前复权（默认） | 曲线连续，无除权跳空，最适合技术分析 | 历史价格可能失真（有时会接近 0） |
| 不复权 | 最新价 = 真实市价 | 除权除息日会有价格跳空，看不出真实趋势 |

---

#### 3.2.5 图表交互

| 交互 | 行为 |
|------|------|
| 鼠标悬停（Hover） | 显示 Crosshair + Tooltip，Tooltip 包含：日期、开/收/高/低、成交量、涨跌幅、所有可见均线的值 |
| 鼠标滚轮 | 横向缩放（Zoom In / Out） |
| 拖拽 | 横向平移（Pan） |
| 触摸板双指 | iPad / 手机端缩放与平移 |
| 双击图表 | 重置到初始视图（最近 3 年 / 60 根 K 线） |
| 点击均线标签 | 显示/隐藏该均线（折叠功能） |

**图表主题**：继承现有终端风格（dark mode），与首页视觉保持一致：
- 背景色：`#090d12`
- 上涨 K 线：`#ff5f5a`（红色）
- 下跌 K 线：`#39d98a`（绿色）
- 网格线：`rgba(116, 156, 190, 0.15)`
- 坐标轴文字：`#8aa0b8`

---

#### 3.2.6 数据加载状态

**首次查询（新冷数据）**：
- 后端首次请求 AkShare 约需 15~45 秒
- 前端显示：`spinner 图标 + "正在从数据源拉取历史数据，首次加载约需 30 秒，请稍候……"`
- 超时策略：请求 60 秒无响应，显示 `error` 状态 + "数据获取超时，请检查网络后重试" + 重试按钮

**缓存命中（热数据）**：
- 响应时间目标 < 2 秒
- 前端显示：轻量 loading（右上角小 spinner，图表区显示旧数据，数据到达后平滑替换）

**缓存更新（盘中/收盘后）**：
- 日线在当日收盘后（15:05~15:30）刷新时，后台异步更新缓存
- 用户下次访问时已是最新数据，无需主动刷新

### 3.3 P1 功能详细说明

#### 3.3.1 股票搜索

**搜索入口**：首页顶部导航栏右侧（刷新按钮左侧），放置一个搜索图标按钮，点击展开搜索框浮层。

**搜索框交互**：
1. 点击搜索图标 → 展开输入框，聚焦光标
2. 用户输入 → 实时过滤候选项（防抖 300ms）
3. 显示匹配结果列表（最多 10 条）
4. 点击某项 → 跳转到 `/stock/[code]?period=daily&adjust=qfq`

**搜索数据来源**：后端 `/api/stocks/search?q=关键词` 接口，返回 `{ items: [{ code, name, market }] }`

**搜索规则**：
- 支持精确匹配股票代码（如 `600519`）
- 支持模糊匹配名称（如输入"茅台"返回 `600519 贵州茅台`）
- 支持拼音首字母（如 `GZMT` → 贵州茅台）—— **此项为 v1 增强，v1 MVP 可跳过**
- 过滤条件：仅返回沪深 A 股（6 位数字代码，以 `600` / `000` / `001` / `002` / `300` / `301` 开头）

**空结果处理**：显示"未找到匹配的股票，请检查代码是否正确"

---

#### 3.3.2 搜索接口 `/api/stocks/search`

**请求**：`GET /api/stocks/search?q=茅台`

**响应**：
```json
{
  "data": [
    { "code": "600519", "name": "贵州茅台", "market": "sh" },
    { "code": "600518", "name": "康美药业", "market": "sh" }
  ],
  "total": 2,
  "updated_at": "2026-06-17T10:00:00",
  "source": "akshare"
}
```

**实现方式**：
1. 后端预存一份全市场 A 股代码表（启动时从 AkShare `stock_info_a_code_name()` 拉取，存 SQLite）
2. 搜索时在 SQLite 中做 `LIKE '%q%' OR name LIKE '%q%'` 查询
3. 该表每日定时更新一次（与收盘刷新任务合并）

---

### 3.4 P2 功能详细说明

#### 3.4.1 自选股收藏

**存储方式**：localStorage（不涉及后端，简单够用）

**数据模型**：
```json
{
  "watchlist": [
    { "code": "600519", "name": "贵州茅台", "added_at": "2026-06-17" },
    { "code": "000001", "name": "平安银行", "added_at": "2026-06-15" }
  ]
}
```

**交互**：
- 个股详情页右上角有「⭐ 收藏 / ★ 已收藏」按钮
- 点击切换收藏状态
- 首页新增一个「我的自选」Tab（与"强势观察"/"风险雷达"平级），展示收藏列表

> 暂不支持多列表/分组（v1 MVP 范围之外）

## 6. 数据模型

### 6.1 新增数据表 — stock_history（历史行情）

```sql
CREATE TABLE IF NOT EXISTS stock_history (
    code        TEXT    NOT NULL,          -- 股票代码，如 '600519'
    period      TEXT    NOT NULL,          -- 'daily' / 'weekly' / 'monthly'
    adjust      TEXT    NOT NULL,          -- 'qfq' / 'none'
    trade_date  TEXT    NOT NULL,          -- 交易日期，YYYY-MM-DD
    open        REAL    NOT NULL,          -- 开盘价
    high        REAL    NOT NULL,          -- 最高价
    low         REAL    NOT NULL,          -- 最低价
    close       REAL    NOT NULL,          -- 收盘价
    volume      INTEGER NOT NULL,          -- 成交量（手）
    amount      REAL    NOT NULL,          -- 成交额（元）
    turnover    REAL    NOT NULL,          -- 换手率（%）
    updated_at  TEXT    NOT NULL,          -- 最后更新时间（ISO8601）
    source      TEXT    NOT NULL,          -- 数据来源，固定 'akshare'
    PRIMARY KEY (code, period, adjust, trade_date)
);

-- 查询优化索引
CREATE INDEX IF NOT EXISTS idx_history_lookup
    ON stock_history(code, period, adjust, trade_date DESC);
```

### 6.2 新增数据表 — stock_info（股票基础信息，用于搜索）

```sql
CREATE TABLE IF NOT EXISTS stock_info (
    code        TEXT    PRIMARY KEY,       -- 股票代码
    name        TEXT    NOT NULL,          -- 股票名称
    market      TEXT    NOT NULL,          -- 市场：'sh' / 'sz'
    list_date    TEXT    NOT NULL,          -- 上市日期
    updated_at  TEXT    NOT NULL           -- 最后更新时间
);

-- 搜索索引（名称模糊查询）
CREATE INDEX IF NOT EXISTS idx_info_name ON stock_info(name);
```

### 6.3 更新数据表 — 无需改动

现有 4 张表（`market_snapshot` / `stock_daily` / `sector_daily` / `daily_report`）保持不变。

## 7. 边界情况与错误处理

### 7.1 边界情况处理矩阵

| 场景 | 判定规则 | 用户感知 | 技术处理 |
|------|---------|---------|---------|
| 股票代码不存在 | AkShare 返回空 / 报错 `STOCK_NOT_FOUND` | 显示错误卡片："未找到股票 [xxxxxx]，请检查代码是否正确" | HTTP 404，返回 `STOCK_NOT_FOUND` |
| 新上市股票（< 30 天） | `list_date` 距今 < 30 天 | 在统计面板上方显示橙色徽标："新股上市，数据有限" | 前端条件渲染 |
| 股票已退市/停牌至今 | `trade_date` 最新记录距今 > 30 天 | 显示警告横幅："该股票已停牌/退市，图表可能不完整" | 前端显示 |
| 长期停牌中间无数据 | 相邻两条数据 `trade_date` 差 > 5 天 | 图表自动跳空（K 线不连续） | lightweight-charts 正常渲染，tooltip 标注"停牌" |
| AkShare 请求超时（> 60s） | requests 超时 | 显示错误卡片 + "数据获取超时，请检查网络后重试" + 重试按钮 | HTTP 408 |
| AkShare 限频（429） | 触发频率限制 | 显示 loading 状态，3 秒后自动重试一次 | 重试逻辑后端处理 |
| 数据库缓存为空（冷启动） | `stock_history` 无该 code 数据 | 显示 `"首次查询，数据正在准备中，预计 30 秒"` | 后端启动异步拉取，前端轮询 |
| 日线数据量 > 800 根 | 回溯数据过多 | 默认只取最近 3 年（720~750 根），超出截断 | `start_date` 自动计算 |
| 前复权历史价格 < 0.01 | 某些低价股前复权后极小值 | 图表正常渲染，但 tooltip 显示 `≈0`，不报异常 | 前端格式化时做兜底 |
| 网络完全断开 | requests 全部失败 | 显示错误卡片："网络异常，请检查连接后重试" | 前端 catch all |

### 7.2 错误状态 UI

所有错误统一使用以下 UI 组件，不再单独设计：

```
┌─────────────────────────────────────┐
│  ⚠️  [错误标题]                      │
│                                     │
│  [错误描述，如"未找到股票 600519"]    │
│                                     │
│  [重新加载] 按钮（primary）           │
│  [返回首页]  链接（text）             │
└─────────────────────────────────────┘
```

## 10. 风险与依赖

### 10.1 主要风险

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| AkShare 接口字段名变动 | 中 | 高 | 在 `fetch_history()` 中用 `row.get("收盘", 0)` 容错写法，不要裸用 `row["收盘"]`；单元测试覆盖主流股票 |
| AkShare 限频（429） | 高 | 中 | 所有 AkShare 调用加全局锁（`threading.Lock`），避免同一进程并发拉同一接口；缓存命中率 > 90% 后风险大幅降低 |
| 首次冷启动超长等待（> 60s） | 低 | 中 | 前端加超时兜底 + 进度提示；后端加请求队列，不并发多个冷启动 |
| SQLite 单文件性能瓶颈 | 低 | 低 | 3 年日线约 750×N 只股票（N=用户关注量），SQLite 完全可支撑；如有需要再迁移 PostgreSQL |
| 用户输入非法股票代码 | 中 | 低 | 前端输入框加正则校验（6 位数字），后端返回 400，UI 显示友好错误 |
